#### Creating One Tracking Shipment

Resource URI

####POST
```
/v2/trackings.json
```


`api_key`: just your api key that you get from aftership website and please keep it private;

`tracking_number`: only one tracking number per request, empty or multiple are not supported;

`courier`: aftership built-in slugs for supporting couriers.

`title`: just your api key that you get from aftership website and please keep it private;

`source`: only one tracking number per request, empty or multiple are not supported;

`smses`: only one tracking number per request, empty or multiple are not supported;

`emails`: aftership built-in slugs for supporting couriers.

`order_id`: just your api key that you get from aftership website and please keep it private;

`order_id_path`: only one tracking number per request, empty or multiple are not supported;

`customer_name`: aftership built-in slugs for supporting couriers.

`origin_country_iso3`: aftership built-in slugs for supporting couriers.

`destination_country_iso3`: just your api key that you get from aftership website and please keep it private;

####example
```
api_key=755a12083dd17420ec4cbc8a3a60016a8d617d08&tracking_number=CL719526662DE&courier=dhl-germany
```

####response

```
{
  "success": false,
  "message": "Tracking number is already exist."
}
```

there are several possiblities for success as false;

* api_key is invalid
* either courier or tracking_number is not provided
* courier is not supported now
* internal error
